Sử dụng mẫu này để giữ Twenty đồng bộ với dữ liệu sản phẩm từ kho dữ liệu (ví dụ: Snowflake, BigQuery, PostgreSQL).

## Cấu trúc Workflow

1. **Trigger**: Theo lịch trình
2. **Mã nguồn**: Truy vấn kho dữ liệu
3. **Mã nguồn** (tùy chọn): Định dạng dữ liệu thành mảng
4. **Iterator**: Lặp qua mỗi sản phẩm
5. **Upsert bản ghi**: Tạo hoặc cập nhật trong Twenty

## Bước 1: Lên lịch Trigger

Đặt workflow chạy với tần suất phù hợp nhu cầu độ mới dữ liệu:
- Mỗi 5 phút cho đồng bộ gần thời gian thực
- Mỗi giờ cho dữ liệu ít quan trọng
- Hàng ngày cho cập nhật theo lô

## Bước 2: Truy vấn kho dữ liệu

Thêm hành động **Mã nguồn** để lấy dữ liệu gần đây:

```javascript
export const main = async () => {
  const intervalMinutes = 10; // Khớp với tần suất lịch trình
  const cutoffTime = new Date(Date.now() - intervalMinutes * 60 * 1000).toISOString();

  // Thay bằng kết nối kho dữ liệu thực tế
  const response = await fetch("https://your-warehouse-api.com/query", {
    method: "POST",
    headers: {
      "Authorization": "Bearer YOUR_API_KEY",
      "Content-Type": "application/json"
    },
    body: JSON.stringify({
      query: `
        SELECT id, name, sku, price, stock_quantity, updated_at
        FROM products
        WHERE updated_at >= '${cutoffTime}'
      `
    })
  });

  const data = await response.json();
  return { products: data.results };
};
```

<Tip>
Lọc theo `updated_at >= X phút trước` để chỉ lấy bản ghi thay đổi gần đây. Điều này giữ đồng bộ hiệu quả.
</Tip>

## Bước 3: Định dạng dữ liệu (Tùy chọn)

Nếu kho dữ liệu trả về dữ liệu cần biến đổi, thêm hành động **Mã nguồn** khác. Các biến đổi phổ biến bao gồm chuyển đổi kiểu, đổi tên trường và dọn dẹp dữ liệu.

### Ví dụ: Dữ liệu người dùng với trường Boolean và Status

```javascript
export const main = async (params: {
  users: any;
}): Promise<object> => {
  const { users } = params;
  const usersFormatted = typeof users === "string" ? JSON.parse(users) : users;

  // Chuyển chuỗi "true"/"false" thành boolean thực
  const toBool = (v: any) => v === true || v === "true";

  return {
    users: usersFormatted.map((user) => ({
      ...user,
      activityStatus: String(user.activityStatus).toUpperCase(),
      isActiveLast30d: toBool(user.isActiveLast30d),
      isActiveLast7d: toBool(user.isActiveLast7d),
      isActiveLast24h: toBool(user.isActiveLast24h),
      isTwenty: toBool(user.isTwenty),
    })),
  };
};
```

### Ví dụ: Dữ liệu sản phẩm với chuyển đổi kiểu

```javascript
export const main = async (params: { products: any }) => {
  const products = typeof params.products === "string"
    ? JSON.parse(params.products)
    : params.products;

  return {
    products: products.map(product => ({
      externalId: product.id,
      name: product.name,
      sku: product.sku,
      price: parseFloat(product.price),        // Chuỗi → Số
      stockQuantity: parseInt(product.stock_quantity),
      isActive: product.status === "active"    // Chuỗi → Boolean
    }))
  };
};
```

### Ví dụ: Định dạng ngày và tiền tệ

```javascript
export const main = async (params: { deals: any }) => {
  const deals = typeof params.deals === "string"
    ? JSON.parse(params.deals)
    : params.deals;

  return {
    deals: deals.map(deal => ({
      ...deal,
      // Chuyển Unix timestamp sang ngày ISO
      closedAt: deal.closed_timestamp
        ? new Date(deal.closed_timestamp * 1000).toISOString()
        : null,
      // Đảm bảo amount là số (loại bỏ ký hiệu tiền tệ)
      amount: parseFloat(String(deal.amount).replace(/[^0-9.-]/g, "")),
      // Chuẩn hóa tên giai đoạn
      stage: deal.stage?.toLowerCase().replace(/_/g, " ")
    }))
  };
};
```

### Biến đổi phổ biến

| Định dạng nguồn | Định dạng đích | Mã |
|---------------|---------------|------|
| `"true"` / `"false"` | `true` / `false` | `v === true \|\| v === "true"` |
| `"123.45"` | `123.45` | `parseFloat(value)` |
| `"active"` | `"ACTIVE"` | `value.toUpperCase()` |
| `1704067200` (Unix) | Ngày ISO | `new Date(v * 1000).toISOString()` |
| `"$1,234.56"` | `1234.56` | `parseFloat(v.replace(/[^0-9.-]/g, ""))` |
| `null` / `undefined` | `""` | `value \|\| ""` |

## Bước 4: Lặp qua sản phẩm

Thêm hành động **Iterator**:
- Đầu vào: `{{code.products}}`

Lặp qua mỗi sản phẩm trong mảng.

## Bước 5: Upsert mỗi bản ghi

Bên trong iterator, thêm hành động **Upsert bản ghi**:

| Cài đặt | Giá trị |
|---------|-------|
| **Đối tượng** | Đối tượng Sản phẩm tùy chỉnh |
| **Khớp theo** | ID bên ngoài hoặc SKU (định danh duy nhất) |
| **Tên** | `{{iterator.item.name}}` |
| **SKU** | `{{iterator.item.sku}}` |
| **Giá** | `{{iterator.item.price}}` |

<Tip>
Sử dụng **Upsert** (cập nhật hoặc tạo) thay vì xây dựng nhánh riêng cho tạo vs. cập nhật. Nhanh hơn để xây dựng và dễ gỡ lỗi hơn.
</Tip>

## Ví dụ trường hợp sử dụng

| Nguồn | Dữ liệu |
|--------|------|
| **Hệ thống ERP** | Danh mục sản phẩm, giá cả, tồn kho |
| **Nền tảng thương mại điện tử** | Đơn hàng, khách hàng, cập nhật sản phẩm |
| **Kho dữ liệu** | Số liệu tổng hợp, dữ liệu làm giàu |
| **Hệ thống tồn kho** | Mức tồn kho, cảnh báo đặt lại |
